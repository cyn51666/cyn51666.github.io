C Sharp对象职责明确原则

2020-10-12 C Sharp 笔记 0 评论
  1. 前后台代码混编的缺点
    1. 程序编写人员必须非常实习后台数据的设计
    2. 业务逻辑复杂时很难查找错误且不利于后期维护
    3. 不符合面向对象的设计思想
  1. 问题解决
    1. 将数据展示代码和数据访问代码分离
    2. 根据当前需要访问的后台实体，创建一个对应的数据访问类
    3. 将对该实体操作的方法封装到对应的访问类中
